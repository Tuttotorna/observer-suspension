# O1 Disagreement Types v0

## Purpose

This document defines the main types of disagreement that can occur between two O1 annotations of the same input.

Its role is not to force identical wording.
Its role is to distinguish:

- acceptable variation
- meaningful divergence
- structural failure

Without this layer, comparison remains too coarse.

---

## Core principle

Two annotations may differ in wording without differing in structure.

What matters is not verbal identity.
What matters is whether the two annotations preserve the same target and expose the same structural dependency class.

The purpose of disagreement typing is to classify where the divergence actually occurs.

---

## Main disagreement classes

### D0 - No meaningful disagreement

Definition:
The two annotations differ only superficially.

Typical signs:
- same target
- same hidden observer privilege
- same structural distortion class
- same relational structure exposed
- minor wording variation only

Status:
Fully acceptable.

Example:
- "fixed reference frame"
- "chosen observational frame"

If both perform the same structural role, disagreement is negligible.

---

### D1 - Precision disagreement

Definition:
Both annotations are acceptable, but one is structurally sharper or more explicit.

Typical signs:
- same target preserved
- same general observer dependence identified
- one annotation names the distortion more precisely
- one reformulation reduces ambiguity better

Status:
Acceptable divergence.

Meaning:
This is not a failure.
It is a difference in annotation quality.

Example:
- one says "reference frame left implicit"
- the other says "relative rest collapsed into absolute stillness"

Both may be valid, but one is stronger.

---

### D2 - Structural emphasis disagreement

Definition:
Both annotations preserve the target, but they emphasize different structural aspects.

Typical signs:
- same target
- same sentence
- different but still relevant hidden assumptions or distortions highlighted
- both remain structurally usable

Status:
Potentially acceptable, but requires care.

Meaning:
This may indicate that the input sentence contains multiple compressions at once.

Example:
One annotation emphasizes frame dependence.
Another emphasizes perceptual mediation.

This is acceptable only if neither abandons the original target.

---

### D3 - Boundary disagreement

Definition:
One annotation remains acceptable, while the other becomes borderline.

Typical signs:
- both appear plausible at first
- one preserves referential continuity more weakly
- one moves closer to theory, abstraction, or interpretation load
- one still passes, but only narrowly

Status:
Important disagreement.

Meaning:
This is where protocol boundaries become visible.

Example:
"I see a tree"
- one annotation keeps observer-signal-object relation explicit
- another shifts toward general perceptual construction language

Both may survive, but not equally well.

---

### D4 - Verdict disagreement

Definition:
One annotation is acceptable and the other is rejected.

Typical signs:
- one preserves target and structural explicitness
- the other drifts into abstraction, rhetoric, or metaphysical language
- O1 gain threshold differs across the two records

Status:
Strong disagreement.

Meaning:
This is not just variation in style.
This is a difference between decomposition and failure.

Example:
- acceptable: "changes position relative to the observer's frame"
- rejected: "motion is an illusion"

---

### D5 - Target disagreement

Definition:
The two annotations are no longer about the same effective target.

Typical signs:
- one remains tied to the original sentence
- the other rewrites the problem into a broader philosophical claim
- referential continuity breaks

Status:
Invalid comparison unless repaired.

Meaning:
Comparison only works if the target is shared.

---

## Disagreement dimensions

When comparing two annotations, inspect divergence along these dimensions:

1. target preservation
2. hidden assumption identification
3. distortion identification
4. reformulation explicitness
5. emergent structure type
6. gain score difference
7. final verdict difference

These dimensions help locate where the disagreement actually sits.

---

## Minimal comparison rule

When two annotations differ, classify them in this order:

1. Is the target still the same?
   - if no, classify as D5

2. Is one annotation rejected and the other accepted?
   - if yes, classify as D4

3. Is one annotation clearly borderline while the other is clearly acceptable?
   - if yes, classify as D3

4. Do both preserve the target but emphasize different structure?
   - if yes, classify as D2

5. Do both expose the same structure, but one is sharper?
   - if yes, classify as D1

6. Otherwise:
   - classify as D0

This ordering prevents soft disagreements from hiding hard failures.

---

## Interpretation rule

A disagreement type does not tell us which annotation is metaphysically true.

It tells us only what kind of structural divergence occurred inside O1.

That is enough for protocol analysis.

---

## Good use

Use disagreement typing to:
- compare two annotations of the same input
- analyze annotation drift across time
- separate acceptable variation from failure
- prepare future inter-annotator studies

---

## Bad use

Do not use disagreement typing to:
- rank intelligence
- judge the author psychologically
- replace the gain criteria
- force verbal uniformity

The point is structural consistency, not stylistic obedience.

---

## Minimal conclusion

O1 comparison becomes stronger once disagreement itself is typed.

Without disagreement types, comparison remains binary.
With disagreement types, comparison becomes structured.
