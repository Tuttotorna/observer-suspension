# O1 Application Template

This document defines the standard application template for Protocol O1.

Its purpose is simple:

**make observer suspension repeatable**

Without a fixed template, O1 risks drifting into free-form commentary.

With a fixed template, O1 can be applied, compared, and evaluated across cases.

---

## Why this template matters

Protocol O1 is only useful if it can be applied in a consistent way.

The template below is designed to:

- standardize analysis
- reduce interpretive drift
- make cases comparable
- allow pattern extraction
- prepare future formalization

This is not yet a mathematical formalism.

It is a controlled descriptive procedure.

---

## Standard O1 application fields

Each O1 application should contain the following fields.

### 1. Standard human statement
The ordinary formulation as commonly expressed.

This field should preserve the original statement as closely as possible.

### 2. Domain
The conceptual domain of the case.

Examples:
- time
- causality
- motion
- identity
- perception
- objecthood

### 3. Implicit privileged observer assumption
The hidden assumption that treats the human observer, local frame, or present-centered access as primary.

Typical forms include:
- local point of view treated as natural
- lived succession treated as primitive
- present access treated as ontological center
- passive observation treated as neutral
- human-readable direction treated as explanatory structure

### 4. Distortion introduced
What becomes compressed, absolutized, hidden, or falsely stabilized by the observer-centered assumption.

Typical distortions include:
- relation presented as intrinsic property
- configuration reduced to event
- dependency reduced to narrative arrow
- local access treated as universal center
- phenomenology treated as ontology

### 5. De-centered reformulation
A reformulation that suspends the privileged observer assumption.

Important:
This does **not** require removing all frames, perspectives, or observational access.

It only requires that no human observer-position be silently elevated into default ontological center.

### 6. Emerging structure
What becomes visible after observer suspension.

Examples:
- relational structure
- frame dependence
- conditional dependency
- local index
- state-ordering
- transition structure
- stabilized configuration
- partial ordering

### 7. O1 outcome class
Each case should receive one of the following outcome labels:

- **Outcome A - trivial reformulation**  
  The de-centered version adds little or no structural value.

- **Outcome B - moderate structural clarification**  
  The reformulation exposes a useful relational or frame-dependent structure.

- **Outcome C - strong structural gain**  
  The reformulation reveals a genuinely more general and promising descriptive structure.

### 8. Pattern tags
Each case should include one or more recurring pattern tags, when appropriate.

Current candidate tags include:

- `absolute_to_relational`
- `phenomenology_to_structure`
- `narrative_arrow_to_dependency`
- `single_center_to_local_index`
- `container_to_transition`
- `readability_to_compression`
- `primitive_object_to_stabilized_configuration`
- `sequence_to_partial_order`

### 9. Notes
Optional field.

Used only when a case requires a short clarification, limitation, or warning.

This field should remain brief.

## Minimal example

### Standard human statement
`Time flows.`

### Domain
`time`

### Implicit privileged observer assumption
- lived succession treated as primitive
- phenomenological continuity projected into world structure

### Distortion introduced
- ordered difference compressed into flow metaphor
- human temporal experience treated as ontological default

### De-centered reformulation
Ordered differences between configurations are observed; "flow" is a local phenomenological description rather than a necessary primitive of the system.

### Emerging structure
- ordering instead of primitive flow
- structure instead of phenomenology

### O1 outcome
`Outcome C`

### Pattern tags
- `phenomenology_to_structure`
- `container_to_transition`

### Notes
- especially strong candidate for deeper formal work

---

## Evaluation rule

An O1 application should be considered successful only if the de-centered reformulation produces at least one of the following:

- clearer relational structure
- less false absolutization
- less narrative compression
- more explicit frame dependence
- more reusable pattern extraction
- stronger descriptive leverage for future formalization

If none of these occur, the case should not be inflated.

It should remain an Outcome A or weak Outcome B.

---

## Limitation

This template does not guarantee truth.

It only controls one specific operation:

**suspending the privileged observer and checking what structural gain follows**

This is a methodological filter, not a final ontology.
