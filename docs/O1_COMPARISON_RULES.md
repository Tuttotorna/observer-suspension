# O1 Comparison Rules v0

## Purpose

This document defines how to compare two O1 annotations written for the same input statement.

Without comparison rules, disagreement is hard to interpret.
With comparison rules, it becomes possible to distinguish:

- acceptable variation
- weak annotation drift
- real structural conflict

These rules do not decide which annotation is true in an absolute sense.
They only define when two annotations are compatible, weakly divergent, or incompatible.

---

## Comparison unit

The comparison unit is:

one original input statement
plus
two O1 records built from that same statement.

The standard formulation must be identical.

If the original input differs, the comparison is invalid.

---

## Core principle

Two annotations do not need identical wording.

They need structural compatibility.

The question is not:

"Did the annotators write the same sentence?"

The real question is:

"Did they expose the same hidden observer privilege, the same kind of distortion, and a compatible decentered structure?"

---

## Comparison dimensions

Two O1 annotations should be compared along five dimensions.

### D1 - Target preservation

Question:
Do both annotations preserve the same original referent or event?

Compatible:
Both still refer to the same target.

Incompatible:
One annotation shifts the target to something broader, narrower, or different.

Example:
Original: The object is still

Compatible pair:
- relative to the chosen frame
- within the selected reference frame

Incompatible pair:
- one remains about the object's position
- the other turns into a claim about reality in general

---

### D2 - Hidden assumption compatibility

Question:
Do both annotations identify the same observer-centered assumption class?

Compatible:
Both identify the same type of hidden privilege, even if phrased differently.

Example:
- assumes a fixed reference frame treated as absolute
- assumes the observer frame is left implicit and treated as neutral

These are compatible.

Incompatible:
One identifies frame privilege, the other identifies something unrelated such as morality, intention, or metaphysics.

---

### D3 - Structural distortion compatibility

Question:
Do both annotations describe the same kind of compression or false stabilization?

Compatible:
Both point to the same structural loss.

Example:
- relative motion collapsed into absolute motion
- frame-dependent motion presented as intrinsic

These are compatible.

Incompatible:
One says the issue is loss of frame dependence, the other says the issue is merely linguistic ambiguity with no structural claim.
