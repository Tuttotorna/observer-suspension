# O1 Benchmark Set v0

This document defines the first compact internal benchmark for Protocol O1.

Its purpose is not to prove the framework.

Its purpose is to test whether O1 produces repeatable structural gain across a small, fixed, comparable case set.

The benchmark is intentionally narrow.

It includes:

- 5 temporal statements
- 5 causal statements

All cases are analyzed using the standard O1 application template.

---

## Why this benchmark exists

At this stage, the repository contains:

- a general framework
- Protocol O1
- first examples
- domain-focused case sets
- first pattern candidates
- a standard application template

The correct next step is not indefinite expansion.

The correct next step is to apply O1 to a stable reference set and test whether:

1. structural gains recur
2. pattern tags recur
3. certain domains are more productive than others
4. O1 yields more than intelligent paraphrase

This benchmark is the first internal test of that question.

# Benchmark A - Time

## A1

### Standard human statement
`Time flows.`

### Domain
`time`

### Implicit privileged observer assumption
- lived succession treated as primitive
- phenomenological continuity projected into world structure

### Distortion introduced
- ordered difference compressed into flow metaphor
- human temporal experience treated as ontological default

### De-centered reformulation
Ordered differences between system configurations are observed; "flow" is a local phenomenological description, not necessarily a primitive feature of the system.

### Emerging structure
- ordering instead of primitive flow
- structure instead of phenomenology

### O1 outcome
`Outcome C`

### Pattern tags
- `phenomenology_to_structure`
- `container_to_transition`

### Notes
- high-yield temporal case

# Benchmark-level observations

Across this initial benchmark, the most recurrent pattern tags are:

- `phenomenology_to_structure`
- `single_center_to_local_index`
- `narrative_arrow_to_dependency`
- `readability_to_compression`

This suggests that Protocol O1 may already be doing more than free-form reformulation.

At minimum, it appears to repeatedly transform:

1. observer-centered immediacy into indexed or relational structure
2. narrative causation into conditioned dependency
3. intuitive readability into compressed description
4. temporal phenomenology into ordered configuration language

---

# Initial benchmark interpretation

This benchmark does not validate O1 as a formal theory.

But it does support the narrower claim that O1 generates recurring descriptive shifts across at least two foundational domains:

- time
- causality

This is enough to justify continued work.

It is not yet enough to justify strong universal claims.

---

# What would count as success for Benchmark v0

Benchmark v0 is successful if it shows all of the following:

1. multiple cases produce Outcome C
2. the same pattern tags recur across domains
3. de-centered reformulations reveal reusable structures
4. O1 improves structural explicitness without collapsing into arbitrary paraphrase

If these conditions continue to hold in later benchmark sets, a stronger formal layer may become justified.
