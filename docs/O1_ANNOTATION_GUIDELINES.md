# O1 Annotation Guidelines v0

## Purpose

This document defines how to write O1 records consistently.

Without annotation rules, the framework becomes too dependent on individual style.
With annotation rules, different records can be compared more cleanly.

These guidelines do not prove correctness.
They only reduce avoidable inconsistency.

---

## Core rule

An O1 record must not sound smarter than the original sentence.

It must become more explicit, not more grandiose.

Good annotation reduces hidden observer privilege.
Bad annotation replaces the original sentence with abstraction.

---

## Field-by-field rules

### 1. Standard formulation

Rule:
Keep the original sentence unchanged.

Do not improve it.
Do not normalize it.
Do not reinterpret it.

Purpose:
Preserve the actual input target.

---

### 2. Hidden observer assumption

Rule:
State the minimal observer-centered assumption required for the sentence to appear natural.

Good:
"Assumes a fixed reference frame treated as absolute."

Bad:
"Humans are biased."

Bad:
"This is naive realism."

Purpose:
Identify the hidden frame, not criticize the speaker.

---

### 3. Structural distortion

Rule:
Describe what is compressed, absolutized, hidden, or falsely stabilized by the assumption.

Good:
"Relative rest is collapsed into absolute stillness."

Bad:
"This is misleading."

Bad:
"This reflects human limitation."

Purpose:
Describe structural loss, not rhetorical weakness.

---

### 4. Decentered reformulation

Rule:
Rewrite the sentence using the weakest sufficient change.

Good:
"The object remains at constant position relative to the chosen reference frame."

Bad:
"Reality is relative."

Bad:
"Nothing is truly still."

Purpose:
Reduce observer privilege while preserving target.

---

### 5. Emergent structure

Rule:
State what relation becomes explicit after decentering.

Good:
"Rest is frame-dependent positional stability."

Bad:
"Reality is deeper than it looks."

Bad:
"Everything depends on perspective."

Purpose:
Name the structural relation now visible.

---

## Acceptance rules

An annotation is acceptable only if all of the following hold:

1. the target is preserved
2. the hidden assumption is explicit
3. the distortion is structural
4. the reformulation is more explicit, not more vague
5. the emergent structure is relational, not rhetorical

---

## Rejection signals

Reject or rewrite a record if it does one or more of the following:

- replaces the target with philosophy
- uses generic phrases like "everything is relative"
- introduces metaphysical claims
- becomes less testable than the original
- criticizes humans instead of describing structure
- sounds deeper while saying less

---

## Good vs bad pattern

### Good pattern
- explicit frame
- explicit relation
- preserved referent
- reduced compression

### Bad pattern
- increased abstraction
- lost referent
- generic relativism
- pseudo-depth

---

## Minimal test before saving a record

Before accepting an O1 record, ask:

1. What is the exact original target?
2. What observer privilege is hidden?
3. What structural compression occurs?
4. Did the rewrite preserve the target?
5. Did the rewrite become more explicit or only more abstract?

If question 5 is not clearly answered, the record is weak.

---

## Boundary

These guidelines do not define truth.

They define annotation discipline.

That is enough for version zero.
