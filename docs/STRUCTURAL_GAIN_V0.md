# tructural Gain v0

This document defines the first working notion of structural gain for Protocol O1.

Its purpose is not to provide a final metric.

Its purpose is to define a minimum discipline for deciding when an O1 reformulation produces real value and when it does not.

At this stage, O1 repeatedly claims to generate structural gain.

Without a sharper definition, that claim remains too soft.

This document exists to harden it.

---

## Why this document is necessary

Protocol O1 already has:

- a conceptual core
- an operational procedure
- focused case sets
- recurring pattern candidates
- an internal benchmark
- a minimal axiomatic layer
- a list of open problems

The central unresolved problem is now clear:

**What counts as structural gain?**

If this is not clarified, O1 risks confusing:

- longer phrasing
- greater caution
- more abstraction
- more technical wording

with genuine structural improvement.

This document defines the first answer.

---

## Minimal definition

A reformulation produces **structural gain** only if it makes explicit a non-trivial structural feature that was hidden, collapsed, or falsely stabilized in the original human statement.

At minimum, the gain must reveal one of the following:

- relation
- dependency
- configuration
- index
- ordering
- transition
- access asymmetry
- frame dependence

If none of these are newly clarified, there is no structural gain.

---

## Core principle

Structural gain is **not** the same as linguistic caution.

Structural gain is **not** the same as descriptive length.

Structural gain is **not** the same as sounding less naive.

Structural gain exists only when the reformulation exposes structure that improves the descriptive organization of the case.

---

## Negative definition

A reformulation does **not** count as structural gain if it does only one or more of the following:

- adds generic qualifiers
- becomes more verbose
- sounds more technical
- weakens the claim without revealing structure
- replaces certainty with vagueness
- substitutes one metaphor with another
- merely avoids commitment

Examples of non-gain:
- "A causes B" -> "A may perhaps in some cases influence B"
- "Time flows" -> "Time seems to move in a complex way"
- "The object is still" -> "The object appears somewhat stable"

These may be softer statements.

They are not necessarily structurally stronger statements.

---

## Positive definition

A reformulation counts as structural gain when it does at least one of the following:

### Gain type 1 - Hidden relation becomes explicit
A property once treated as intrinsic becomes explicitly relational.

Example:
- "The object is still"
- becomes
- "Within the selected frame, the relative positional variation is negligible."

### Gain type 2 - Hidden dependence becomes explicit
A statement once presented as simple event or property becomes dependent on a condition, frame, or configuration.

Example:
- "The past is fixed"
- becomes
- "Certain prior state-relations are no longer alterable within the accessible transition structure."

### Gain type 3 - Narrative compression becomes dependency structure
A human-readable causal story is unpacked into conditional or distributed dependency.

Example:
- "A causes B"
- becomes
- "B emerges under a configuration in which A participates as one active dependency under given conditions."

### Gain type 4 - Local center becomes index
A privileged descriptive center is reduced to local access or local indexing.

Example:
- "The present is now"
- becomes
- "Now designates a local access index within a descriptive frame."

### Gain type 5 - Phenomenology becomes structure
A lived experience is reformulated as ordering, transition, metric relation, or configuration structure.

Example:
- "Time flows"
- becomes
- "Ordered differences appear between configurations; flow is a local phenomenological compression."

### Gain type 6 - Container metaphor becomes transition organization
A vague background concept is replaced by explicit organization of changes.

Example:
- "Things unfold over time"
- becomes
- "The system undergoes distinguishable transformations across ordered configurations."

---

## Working test for structural gain

A candidate O1 reformulation should be tested with the following question:

**What structure is explicit here that was not explicit before?**

If the answer is unclear or empty, the gain is weak or absent.

A stronger secondary test is:

**Can the new explicit structure be reused across cases?**

If yes, the gain is stronger.

---

## Three levels of gain

At this stage, structural gain may be classified in three levels.

### Level 0 - No gain
The reformulation is only:
- longer
- softer
- more careful
- more abstract

but does not expose new structure.

### Level 1 - Local gain
The reformulation exposes a hidden relation, dependency, or index in that specific case.

This is useful, but may remain local.

### Level 2 - Reusable gain
The reformulation exposes a structure that:
- is explicit
- is non-trivial
- recurs across cases
- supports pattern extraction

This is the most important kind of gain for O1.

---

## Structural gain and O1 outcome classes

Structural gain should guide O1 outcome assignment.

### Outcome A
No structural gain or only cosmetic reformulation.

### Outcome B
Local structural gain.
A hidden relation, dependency, frame, or index becomes explicit, but without strong reuse value yet.

### Outcome C
Reusable structural gain.
The reformulation exposes a structure that is not only clearer in the case, but also promising for pattern extraction, comparison, or further formalization.

---

## Gain criteria table

| Criterion | No gain | Local gain | Reusable gain |
|---|---|---|---|
| More words | yes | possible | possible |
| More caution | yes | possible | possible |
| New explicit relation | no | yes | yes |
| New explicit dependency | no | yes | yes |
| New explicit index/frame | no | yes | yes |
| Pattern reuse potential | no | weak/unknown | strong |
| Supports stronger formalization | no | limited | plausible |

## Examples

## Example 1 - no gain

### Human statement
`A causes B.`

### Reformulation
`A may possibly sometimes influence B in certain ways.`

### Evaluation
- more cautious: yes
- more explicit structure: no
- gain: no

### Result
`Level 0 - no gain`

---

## Example 2 - local gain

### Human statement
`The object is still.`

### Reformulation
`Within the selected frame, the relative positional variation of the object is negligible.`

### Evaluation
- frame dependence explicit: yes
- relation explicit: yes
- reusable pattern: possible

### Result
`Level 1 - local gain`

---

## Example 3 - reusable gain

### Human statement
`Time flows.`

### Reformulation
`Ordered differences between configurations are observed; flow is a local phenomenological compression.`

### Evaluation
- phenomenology replaced by structure: yes
- ordering explicit: yes
- reusable across temporal cases: yes

### Result
`Level 2 - reusable gain`

---

## Example 4 - reusable gain

### Human statement
`The event happened because of X.`

### Reformulation
`Event E appears under a configuration in which X is one active dependency among other enabling conditions and interactions.`

### Evaluation
- narrative explanation replaced by dependency structure: yes
- conditions explicit: yes
- reusable across causal cases: yes

### Result
`Level 2 - reusable gain`

---

## Strong warning

Structural gain must not be confused with anti-human phrasing.

A statement is not stronger merely because it sounds colder, less intuitive, or less anthropic.

O1 is not trying to reward alien wording.

It is trying to reward explicit structure.

This distinction is critical.

---

## First operational rule

From this point forward, every O1 application should be implicitly checked against the following rule:

**If the de-centered reformulation does not reveal a new explicit structural feature, it should not be scored as meaningful gain.**

This is the first anti-inflation rule of the framework.

---

## First unresolved issue

Even with this document, structural gain is still not fully formalized.

Important unresolved questions remain:

- Can gain be scored numerically?
- Can gain be compared across domains?
- Can reuse potential be operationalized?
- Can evaluators be constrained more strongly?

This document does not solve those problems.

It only provides the first disciplined threshold.

---

## Minimal conclusion

Structural gain is the central criterion that prevents O1 from collapsing into elegant paraphrase.

At this stage, the minimum acceptable definition is:

**A reformulation has structural gain only if it makes hidden structure explicit in a way that is either locally clarifying or recurrently reusable.**

This is not the final theory of gain.

It is the first necessary filter.
