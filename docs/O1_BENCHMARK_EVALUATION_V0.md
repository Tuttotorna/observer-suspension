# O1 Benchmark Evaluation v0

This document evaluates the initial O1 benchmark using the working notion of structural gain defined in `STRUCTURAL_GAIN_V0.md`.

Its purpose is not to prove O1.

Its purpose is to test whether the benchmark cases actually satisfy the framework's own anti-inflation criteria.

This is the first internal validation step of the repository.

---

## Why this document matters

The repository now contains:

- Protocol O1
- focused case sets
- pattern candidates
- a benchmark
- a minimal axiomatic layer
- a first working notion of structural gain

The next correct question is:

**Do the benchmark cases still hold when judged under the framework's own gain criteria?**

If yes, O1 gains internal consistency.

If not, O1 must correct itself before expanding further.

---

## Structural gain levels used here

This evaluation uses the three-level distinction from `STRUCTURAL_GAIN_V0.md`:

- **Level 0 - no gain**  
  Longer, softer, or more careful wording without explicit structural improvement.

- **Level 1 - local gain**  
  A hidden relation, dependency, frame, or index becomes explicit in that case.

- **Level 2 - reusable gain**  
  The reformulation reveals structure that is explicit, non-trivial, recurrent, and useful for pattern extraction or future formalization.

## Preliminary interpretation

Under the current working notion of structural gain, the benchmark remains strong.

This does not prove O1 is correct in any final sense.

But it does show that the benchmark is not collapsing under the anti-inflation criteria introduced later.

This is important.

It means the framework has, at minimum, retained internal coherence across:

- protocol
- benchmark
- gain definition

---

## Strongest cases

The strongest benchmark cases under current evaluation appear to be:

- A1 - Time flows
- A4 - Events happen in sequence
- A5 - The present is now
- B1 - A causes B
- B3 - The system failed because component Y broke

These cases are especially strong because they do all of the following:

1. expose hidden structure explicitly
2. support recurring pattern tags
3. generalize beyond the individual sentence
4. open plausible routes toward stronger formalization

---

## Weakest case

The weakest benchmark case at present is:

- B5 - Fear causes violence

This does not make it useless.

It only means that its structural gain is more local and less clearly reusable under the current criteria.

This is exactly the kind of internal distinction the repository needs.

---

## First validation result

The first internal validation result is therefore:

**O1 benchmark v0 survives its first gain-based evaluation.**

That is not a proof of discovery.

But it is enough to justify continued work.
